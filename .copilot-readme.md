# Radiohead RAG - Copilot Context Guide

## Project Overview
This is a Retrieval-Augmented Generation (RAG) application that enables semantic search and question-answering over a collection of Radiohead music reviews from Pitchfork. The system uses vector embeddings, semantic similarity search, and OpenAI's LLM to provide context-aware answers about Radiohead albums and tracks.

## Architecture & Workflow

### 1. **Data Pipeline** (`create_database.py`)
- **Purpose**: Transform raw markdown documents into embeddings and store in vector database
- **Process**:
  1. Load all `.md` files from the `data/` directory using `DirectoryLoader`
  2. Split documents into overlapping chunks (1000 chars, 500 char overlap) using `RecursiveCharacterTextSplitter`
  3. Generate vector embeddings using OpenAI's embedding model
  4. Store embeddings in Chroma database (SQLite backend) at `chroma/` directory
- **Key Functions**:
  - `load_documents()`: Loads markdown files
  - `split_text()`: Chunks documents with overlap for context preservation
  - `save_to_chroma()`: Creates and persists vector database
- **Important**: Clears existing `chroma/` directory before rebuilding to ensure fresh embeddings

### 2. **Query/Chat Interface** (`query_data.py`)
- **Purpose**: Answer user questions using semantic search + LLM context
- **Process**:
  1. Accept user query as command-line argument
  2. Perform similarity search against embeddings (returns top 4 results)
  3. Apply relevance threshold (score > 0.7) to filter low-quality matches
  4. Construct prompt with retrieved context using `ChatPromptTemplate`
  5. Send prompt to OpenAI ChatGPT model for response generation
  6. Return response with source attribution (source file paths)
- **Key Variables**:
  - `CHROMA_PATH`: Points to vector database directory
  - `PROMPT_TEMPLATE`: Controls system prompt and context injection
  - Similarity score threshold: 0.7 (configurable)
  - `k=4`: Number of retrieved chunks (in code, appears as `f=4`)

### 3. **Data Collection** (`markdown_script.py`)
- **Purpose**: Download and convert web pages (Pitchfork reviews) to markdown
- **Features**:
  - Downloads webpages using `requests`
  - Converts HTML to markdown using `html2text` and BeautifulSoup
  - Cleans filenames from URLs for organized storage
  - Preserves links and removes navigation/footer clutter
- **Note**: Creates the markdown files in `data/` that feed into the RAG pipeline

## Common Issues & Solutions

### Issue: Low quality or no results
- **Cause**: Query similarity score below 0.7 threshold
- **Solution**: Modify line in `query_data.py` where `results[0][1] < 0.7`
- **Prevention**: Ensure database is fresh (run `create_database.py`)

### Issue: API errors
- **Cause**: Missing or invalid `OPENAI_API_KEY`
- **Solution**: Verify `.env` file exists and key is valid

### Issue: Outdated embeddings
- **Cause**: `create_database.py` not re-run after adding new markdown files
- **Solution**: Re-run `python create_database.py` to regenerate embeddings


## Key Tuning Parameters

- **Chunk size** (`query_data.py`): 1000 characters
- **Chunk overlap**: 500 characters (helps preserve context at boundaries)
- **Similarity threshold**: 0.7 (minimum relevance score)
- **Number of results**: k=4 (number of chunks retrieved)
- **Model**: `ChatOpenAI()` (uses gpt-3.5-turbo by default)
- **Embeddings**: `OpenAIEmbeddings()` (uses text-embedding-3-small by default)
